# 📱 دليل نشر «بسمة» على Google Play

جهزت لك كل الملفات المطلوبة. اتبع الخطوات بالترتيب.

## المرحلة ١ — تحديث الموقع (ريبو BASMAH)

ارفع هذه الملفات إلى ريبو `hyr55cc/BASMAH` (استبدل القديم):

| الملف | وش تغيّر |
|---|---|
| `index.html` | زر 🚩 إبلاغ لكل دعاء + فلترة الكلمات المسيئة والروابط والأرقام + حد زمني دقيقتين بين الأدعية + إخفاء الأدعية المبلَّغ عنها (٣ بلاغات فأكثر) + رابط سياسة الخصوصية في الإعدادات — **مطلوب لسياسة UGC في قوقل** |
| `privacy.html` | صفحة سياسة الخصوصية — **إلزامية، بدونها يُرفض التطبيق** |
| `manifest.json` | أيقونات حقيقية 192/512 + maskable + خاصية id |
| `icons/` (٤ ملفات) | أيقونات مضبوطة المقاسات |
| `sw.js` | إضافة الملفات الجديدة للكاش (v5) |
| `firestore.rules` | **لا يُرفع للريبو** — انسخه في Firebase Console → Firestore → Rules → Publish (يمنع التخريب في قاعدة البيانات) |

تأكد بعد الرفع أن هذا الرابط يفتح: `https://hyr55cc.github.io/BASMAH/privacy.html`

## المرحلة ٢ — بناء حزمة الأندرويد (PWABuilder)

1. افتح **pwabuilder.com** وأدخل: `https://hyr55cc.github.io/BASMAH/`
2. اضغط **Package for Stores → Android → Google Play**
3. الإعدادات:
   - **Package ID:** `io.github.hyr55cc.basmah`
   - **App name:** بسمة — أذكار الصباح والمساء / **Short name:** بسمة
   - **Version:** 1.0.0
   - **Signing key:** اختر *Create new* — **⚠️ احفظ ملف `signing.keystore` وكلمات المرور في مكان آمن، ضياعها يعني عدم قدرتك على التحديث مستقبلاً**
   - في All Settings: فعّل **Location delegation** (لزر 📍 موقعي) و **Notification delegation**
4. حمّل الملف — بيعطيك `.aab` (هذا اللي يترفع) + `assetlinks.json` جاهز فيه بصمة التوقيع

## المرحلة ٣ — ربط النطاق (أهم خطوة تقنية!)

بدونها يظهر شريط المتصفح داخل التطبيق ويُرفض غالباً لأنه «مجرد موقع».

1. أنشئ ريبو جديد باسم: **`hyr55cc.github.io`** (بالضبط هذا الاسم)
2. فعّل GitHub Pages فيه
3. ضع فيه مجلد `.well-known` وبداخله ملف `assetlinks.json` (خذ الملف اللي أعطاك إياه PWABuilder — جهزت لك قالب في مجلد `well-known`)
4. بعد أول رفع للتطبيق في Play Console: روح **Setup → App integrity → App signing** وانسخ بصمة **SHA-256** وأضفها كسطر ثاني في نفس الملف (قوقل يعيد توقيع التطبيق، فلازم البصمتين)
5. تأكد أن الرابط يفتح: `https://hyr55cc.github.io/.well-known/assetlinks.json`

## المرحلة ٤ — Play Console

**إنشاء التطبيق:** All apps → Create app → الاسم: بسمة — أذكار الصباح والمساء → App → Free

### إجابات نموذج App content (عشان ما يُرفض):

| السؤال | الإجابة |
|---|---|
| Privacy policy | `https://hyr55cc.github.io/BASMAH/privacy.html` |
| App access | All functionality is available without special access (ما فيه تسجيل دخول) |
| Ads | No, my app does not contain ads |
| Content rating | استبيان IARC → فئة Utility/Productivity → **نعم للسؤال عن تفاعل المستخدمين/مشاركة المحتوى (بسبب جدار الأدعية)** → البقية لا |
| Target audience | 13 سنة فأكبر (لا تختار أقل من 13 عشان لا تدخل بسياسات الأطفال المعقدة) |
| News app | No |
| Data safety | ↓ بالجدول التالي |

### نموذج Data safety (مهم جداً — التصريح الخاطئ = رفض):

- **Does your app collect or share data?** → Yes
- **Location → Approximate location:** يُجمع، اختياري، لأغراض App functionality، لا يُشارك، يُعالج لحظياً ولا يُخزن
- **App activity → Other user-generated content:** يُجمع (نص الدعاء)، اختياري، App functionality، **يُشارك** (يظهر لبقية المستخدمين)، يُحذف تلقائياً خلال 24 ساعة
- **Device or other IDs:** يُجمع (رمز إشعارات FCM)، App functionality
- **Is data encrypted in transit?** → Yes (كل الاتصالات HTTPS)
- **Can users request data deletion?** → Yes (عبر البريد المذكور في سياسة الخصوصية)

### صفحة المتجر (Store listing):

- **الوصف القصير (80 حرف):** أذكار الصباح والمساء ومواقيت الصلاة والمسبحة وسورة الكهف — بدون إعلانات
- **الوصف الطويل:** اذكر الميزات: أذكار الصباح/المساء/النوم/الاستيقاظ وبعد الصلاة، مواقيت الصلاة لمدن السعودية والعالم، المسبحة الإلكترونية، أسماء الله الحسنى، سورة الكهف، جدار الأدعية، الوضع الليلي، يعمل بدون إنترنت. **تجنب:** كلمات مثل «أفضل تطبيق» أو «رقم ١» أو حشو كلمات مفتاحية — سبب شائع للرفض (Metadata policy)
- **الأيقونة:** `icons/icon-512.png`
- **Feature graphic:** `feature-graphic-1024x500.png` (جاهز بالمجلد)
- **لقطات الشاشة:** افتح الموقع بجوالك وخذ 4-6 لقطات (الرئيسية، الأذكار، مواقيت الصلاة، المسبحة، الإعدادات)
- **اللغة الافتراضية:** العربية (السعودية) ar-SA

## المرحلة ٥ — الاختبار المغلق ثم النشر (إلزامي لحسابك الشخصي)

حسابك شخصي، فقوقل تشترط قبل النشر العام:

1. **Testing → Closed testing → Create track** → ارفع ملف `.aab`
2. أضف قائمة إيميلات المختبرين وشارك رابط الانضمام
3. تحتاج **12 مختبِر على الأقل** يظلون منضمين **14 يوم متواصل** (عائلة، أصدقاء، قروبات)
4. بعد 14 يوم: **Dashboard → Apply for production** وجاوب الأسئلة (وش تعلمت من الاختبار، جمهورك المستهدف، إلخ)
5. الموافقة عادة خلال 7 أيام، بعدها انشر في Production 🎉
